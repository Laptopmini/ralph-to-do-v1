# PRD: Todo Logic & Data Layer

## Objective

Implement a pure TypeScript module (`src/todo.ts`) that defines the `Todo` data type and provides all CRUD operations and LocalStorage persistence for to-do items. This module must have zero DOM dependencies so it can be fully exercised by Jest unit tests.

## Context

The app is a static vanilla HTML/CSS/TypeScript site served by `serve` on port 3000. There is no framework. All logic lives in `src/`, Jest unit tests live in `tests/unit/`, and TypeScript is compiled via SWC for Jest. This ticket establishes the data layer that every other part of the application will import — it must be a standalone, pure module. LocalStorage is the only persistence mechanism; the key `"todos"` stores a JSON-serialized array of `Todo` objects. Each `Todo` item carries a UUID generated by `crypto.randomUUID()`, a text description, and a completed boolean.

## Constraints

- Must be a pure module with no DOM dependencies so it can be unit-tested with Jest
- Must use `localStorage` for persistence with the key `"todos"`
- All functions must be named exports

## Tasks

- [ ] Define Todo interface and storage key. Create `src/todo.ts` and add: (1) an exported `interface Todo` with fields `id: string`, `text: string`, and `completed: boolean`; (2) an exported constant `STORAGE_KEY = "todos"` of type `string`. Do not add any other code in this task. `[test: npx jest tests/unit/define-todo-interface-storage-key.test.ts]`
- [ ] Implement loadTodos function. In `src/todo.ts`, add an exported function `loadTodos(): Todo[]` that reads `localStorage.getItem(STORAGE_KEY)`, parses the result as JSON, and returns the parsed array. If the key is missing (returns `null`), the value is not valid JSON (throws on `JSON.parse`), or the parsed value is not an array (`!Array.isArray`), return `[]` instead. `[test: npx jest tests/unit/implement-loadtodos-function.test.ts]`
- [ ] Implement saveTodos function. In `src/todo.ts`, add an exported function `saveTodos(todos: Todo[]): void` that calls `JSON.stringify(todos)` and writes the result to `localStorage.setItem(STORAGE_KEY, ...)`. The function has no return value. `[test: npx jest tests/unit/implement-savetodos-function.test.ts]`
- [ ] Implement addTodo function. In `src/todo.ts`, add an exported function `addTodo(todos: Todo[], text: string): Todo[]`. If `text.trim()` is empty, return `todos` unchanged. Otherwise create a new `Todo` object: `id` set to `crypto.randomUUID()`, `text` set to `text.trim()`, `completed` set to `false`. Return a new array with the new item prepended: `[newTodo, ...todos]`. Do not call `saveTodos` — persistence is the caller's responsibility. `[test: npx jest tests/unit/implement-addtodo-function.test.ts]`
- [ ] Implement toggleTodo function. In `src/todo.ts`, add an exported function `toggleTodo(todos: Todo[], id: string): Todo[]`. Return a new array produced by mapping over `todos`: for the item whose `id` matches the argument, return a copy with `completed` flipped (`!item.completed`); for all other items, return them unchanged. If no item matches `id`, return the original array unchanged. Do not call `saveTodos`. `[test: npx jest tests/unit/implement-toggletodo-function.test.ts]`
- [ ] Implement deleteTodo function. In `src/todo.ts`, add an exported function `deleteTodo(todos: Todo[], id: string): Todo[]`. Return a new array containing only the items whose `id` does not equal the argument (i.e., `todos.filter(item => item.id !== id)`). If no item matches `id`, the returned array equals the original. Do not call `saveTodos`. `[test: npx jest tests/unit/implement-deletetodo-function.test.ts]`
